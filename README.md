# LibraryBotTG — Telegram-бот библиотекарь

Бот для поиска книг, распознавания по обложке и цитате, получения рекомендаций и ссылок на скачивание в разных форматах.

---

## Возможности

- **Поиск книг** — по названию или автору (Google Books API). Выбор книги из списка, карточка с описанием, рейтингом, обложкой.
- **Форматы** — кнопки FB2, EPUB, PDF, TXT, Аудио: поиск файла в открытых источниках и отправка в чат.
- **По обложке** — отправьте фото обложки книги: бот распознаёт текст (Google Vision API) и ищет книгу.
- **По цитате** — отправьте цитату из книги: бот определяет произведение (Gemini API) и показывает карточку.
- **Похожие книги** — на карточке книги кнопка «Похожие книги»: рекомендации через Gemini и ссылки на покупку (Ozon, Wildberries, Litres).

---

## Инструкция по использованию (для пользователей бота)

### Запуск диалога

1. Найдите бота в Telegram и нажмите **«Запустить»** или отправьте команду `/start`.
2. Появится приветствие и **главное меню** с кнопками:
   - **Найти книгу**
   - **По обложке**
   - **По цитате**
3. В любой момент можно нажать **«Назад»**, чтобы вернуться в главное меню.

---

### Поиск книги по названию или автору

1. Нажмите **«Найти книгу»**.
2. Введите запрос: название книги или имя автора (например: `Булгаков`, `Мастер и Маргарита`).
3. Бот покажет список найденных книг. Нажмите на нужную — откроется **карточка книги**.
4. На карточке есть:
   - обложка, название, автор, описание, рейтинг;
   - кнопки форматов: **FB2**, **EPUB**, **PDF**, **TXT**, **Аудио** — при нажатии бот попытается найти файл и отправить его;
   - ссылки **Ozon**, **Wildberries**, **Litres** — поиск книги в магазине;
   - кнопка **«Похожие книги»** — рекомендации по похожим произведениям.

---

### Определение книги по фото обложки

1. Нажмите **«По обложке»**.
2. Отправьте **фото обложки книги** (желательно чёткое, с читаемым названием и автором).
3. Бот распознает текст на изображении и найдёт книгу, затем покажет её карточку. Если не получится — появится сообщение об ошибке, можно попробовать другое фото.

---

### Определение книги по цитате

1. Нажмите **«По цитате»**.
2. Отправьте **текст цитаты** из книги (скопируйте или напишите вручную).
3. Бот по цитате определит произведение и покажет карточку книги. Если не уверен — сообщит об этом.

---

### Похожие книги и открытие карточки

1. На карточке любой книги нажмите **«Похожие книги»**.
2. Бот пришлёт список из нескольких похожих книг (короткие карточки с названием и автором).
3. У каждой рекомендации есть кнопка **«Открыть»** — по нажатию откроется полная карточка этой книги (описание, форматы, магазины, снова «Похожие книги»).

---

### Важно

- **Файлы книг** (FB2, EPUB, PDF и т.д.) ищутся в открытых источниках; их распространение может противоречить авторскому праву. Используйте ответственно.
- Для поиска и рекомендаций нужен доступ к интернету и корректная настройка API на стороне владельца бота.

## Подготовка к загрузке в Git

- В репозиторий **не попадают** файлы с токенами и секретами: `.gitignore` исключает `.env`, `data/`, `*.db` и другие чувствительные пути.
- В репозитории есть только **`.env.example`** — шаблон переменных без значений. Перед запуском скопируйте его в `.env` и заполните ключи локально.
- Перед первым `git push` убедитесь, что файл `.env` не добавлен в коммит: `git status` не должен показывать `.env`.

---

## Рекомендации по запуску

- **Виртуальное окружение** — используйте `venv` или `virtualenv`, чтобы не смешивать зависимости с системным Python.
- **Версия Python** — нужен Python 3.11 или выше. Проверка: `python --version`.
- **Токены заранее** — получите все три ключа (Bot, Google, Gemini) до первого запуска; без них поиск и распознавание не заработают.
- **Запуск из корня проекта** — всегда запускайте `python bot.py` из папки `LibraryBotTG`, где лежат `bot.py` и `config.py`.
- **Логи** — при ошибках смотрите вывод в консоли; при необходимости включите более подробное логирование в `bot.py`.
- **Порт и сеть** — бот использует long polling (исходящие запросы к Telegram), открывать входящий порт не нужно. Доступ в интернет обязателен.

## Пошаговый запуск

### Шаг 1. Установить Python 3.11+

Проверьте версию:

```bash
python --version
```

Если Python нет или версия старая — установите с [python.org](https://www.python.org/downloads/) или через менеджер пакетов ОС.

---

### Шаг 2. Получить токены

| Токен | Где взять |
|-------|-----------|
| **BOT_TOKEN** | Telegram → [@BotFather](https://t.me/BotFather) → `/newbot` → введите имя и username бота → скопируйте выданный токен. |
| **GOOGLE_API_KEY** | [Google Cloud Console](https://console.cloud.google.com) → создайте проект (или выберите существующий) → «API и сервисы» → «Включить API» → включите **Books API** и **Cloud Vision API** → «Учётные данные» → «Создать учётные данные» → «API-ключ». |
| **GEMINI_API_KEY** | [Google AI Studio](https://aistudio.google.com) → войдите в аккаунт Google → «Get API key» / «Создать API-ключ» → скопируйте ключ. |

---

### Шаг 3. Склонировать или скачать проект

Если проект в Git:

```bash
git clone <URL вашего репозитория>
cd LibraryBotTG
```

Либо распакуйте архив с проектом и откройте папку в терминале.

---

### Шаг 4. Создать виртуальное окружение (рекомендуется)

```bash
python -m venv .venv
```

Активация:

- **Windows (cmd):** `.venv\Scripts\activate.bat`
- **Windows (PowerShell):** `.venv\Scripts\Activate.ps1`
- **Linux / macOS:** `source .venv/bin/activate`

После активации в начале строки появится `(.venv)`.

---

### Шаг 5. Установить зависимости

```bash
pip install -r requirements.txt
```

---

### Шаг 6. Настроить переменные окружения

Создайте файл `.env` в корне проекта (рядом с `bot.py`):

- Скопируйте шаблон:  
  **Windows:** `copy .env.example .env`  
  **Linux / macOS:** `cp .env.example .env`
- Откройте `.env` в редакторе и вставьте свои значения:

```
BOT_TOKEN=ваш_токен_от_BotFather
GOOGLE_API_KEY=ваш_ключ_google_cloud
GEMINI_API_KEY=ваш_ключ_gemini
```

Сохраните файл. Файл `.env` не должен попадать в Git.

---

### Шаг 7. Запустить бота

Из корня проекта (с активированным при необходимости `.venv`):

```bash
python bot.py
```

В консоли должно появиться сообщение вроде «Бот запущен.» — после этого откройте бота в Telegram и нажмите «Запустить» или отправьте `/start`.

Остановка: в терминале нажмите `Ctrl+C`.

---

### Шаг 8. (По желанию) Запустить тесты

```bash
pip install -r requirements-dev.txt
python -m pytest tests/ -v
```

Подробнее — в [MANUAL_TESTING.md](MANUAL_TESTING.md).

## Деплой на Railway

Бот использует **long polling** (сам ходит к Telegram), поэтому на Railway не нужен публичный URL и веб-сервер — достаточно одного сервиса из GitHub с переменными окружения.

### 1. Аккаунт и проект

1. Зайдите на [railway.app](https://railway.app) и войдите через **GitHub**.
2. Нажмите **New Project** → **Deploy from GitHub repo**.
3. Выберите репозиторий **ChillLibraryTgBot** (или свой форк). Railway подхватит корень репозитория.

### 2. Сборка и запуск

- Railway по умолчанию обнаружит **Dockerfile**: соберёт образ и запустит `python bot.py`.
- Если репозиторий без Dockerfile, в настройках сервиса укажите:
  - **Build Command:** `pip install -r requirements.txt`
  - **Start Command:** `python bot.py`
  - **Root Directory:** оставьте пустым (корень репозитория).

### 3. Переменные окружения (обязательно)

В проекте Railway откройте ваш сервис → вкладка **Variables** и добавьте:

| Переменная         | Описание |
|--------------------|----------|
| `BOT_TOKEN`        | Токен от [@BotFather](https://t.me/BotFather). |
| `GOOGLE_API_KEY`   | Ключ Google Cloud (Books API, при необходимости Vision). |
| `GEMINI_API_KEY`   | Ключ из [Google AI Studio](https://aistudio.google.com) для цитат и рекомендаций. |
| `GOOGLE_VISION_KEY`| (по желанию) Отдельный ключ для Vision; если пусто, используется `GOOGLE_API_KEY`. |
| `GOOGLE_CSE_CX`    | (по желанию) ID поисковой машины для fallback-поиска файлов. |

После сохранения переменных Railway пересоберёт/перезапустит сервис. **Токены в репозиторий не добавляйте** — только в Variables.

### 4. Деплой и логи

- После **Deploy** в логах должно появиться сообщение вроде «Бот запущен.».
- Если бот не отвечает: проверьте **Deploy Logs** на ошибки и что все переменные заданы (в т.ч. `BOT_TOKEN`).
- База SQLite (`data/`) живёт внутри контейнера; при новом деплое или пересоздании сервиса данные сбросятся. Для постоянного хранения можно позже подключить Volume (см. документацию Railway).

### 5. Домен и порт

Для long polling **открывать порт и настраивать домен не нужно**. Бот сам обращается к API Telegram. В настройках сервиса можно отключить генерацию публичного домена, если он не нужен.

## Деплой через Docker (локально)

- Соберите образ: `docker build -t chilllibrary-bot .`
- Запуск с переменными:  
  `docker run --env-file .env chilllibrary-bot`  
  (файл `.env` только локально, в образ не копируется — см. `.dockerignore`.)
- Том для `data/` при необходимости: `-v $(pwd)/data:/app/data`

**Важно:** ни в Docker-образ, ни в Git не помещайте реальные токены — только через Variables/Environment или `--env-file`.

## Лицензия и ограничения

- Код можно использовать и дорабатывать по своему усмотрению.
- Отправка файлов книг через бота может нарушать авторские права; ответственность лежит на владельце бота и пользователях.
- Telegram может ограничивать или блокировать ботов при жалобах на контент.
