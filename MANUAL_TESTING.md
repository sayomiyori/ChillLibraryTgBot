# Тестирование бота

## Автотесты (pytest)

Установка зависимостей для тестов:

```bash
pip install -r requirements-dev.txt
```

Запуск всех тестов:

```bash
python -m pytest tests/ -v
```

Кратко: `tests/test_database.py` — БД; `tests/test_services_*.py` — сервисы (Books, Vision, Gemini, Files) с моками; `tests/test_handlers.py` — форматирование карточек и клавиатуры.

---

## Ручное тестирование

Перед запуском убедись, что в `.env` заданы `BOT_TOKEN`, `GOOGLE_API_KEY`, `GEMINI_API_KEY`. Запуск: `python bot.py`.

## 1. Старт и навигация

| # | Действие | Ожидание |
|---|----------|----------|
| 1.1 | Отправить `/start` | Приветствие и главное меню: «Найти книгу», «По обложке», «По цитате». |
| 1.2 | Нажать «Назад» | То же, что после `/start` — главное меню. |

---

## 2. Поиск книг

| # | Действие | Ожидание |
|---|----------|----------|
| 2.1 | Нажать «Найти книгу» | Сообщение «Введи название книги или автора для поиска», кнопка «Назад». |
| 2.2 | Ввести запрос (например, «Булгаков Мастер») | Сообщение «Ищу…», затем список книг с кнопками выбора. |
| 2.3 | Нажать на одну из книг в списке | Карточка книги: обложка (если есть), название, автор, описание, рейтинг, кнопки форматов (FB2, EPUB, PDF, TXT, Аудио), ссылки Ozon/WB/Litres, кнопка «Похожие книги». |
| 2.4 | Ввести пустой или только пробелы | Сообщение с просьбой ввести текст для поиска. |
| 2.5 | Ввести несуществующий запрос (например, «xyznonexistent123») | Сообщение «По запросу ничего не найдено». |

## 6. Похожие книги (рекомендации)

| # | Действие | Ожидание |
|---|----------|----------|
| 6.1 | Открыть любую книгу → нажать «Похожие книги» | Сообщение «Ищу похожие книги…», затем заголовок «Похожие книги:» и до 5 коротких карточек (название, автор, кнопки «Открыть», Ozon, Litres). |

| 6.3 | Если API недоступен или ошибка | Сообщение «Не удалось получить рекомендации» или «Похожих книг не найдено». |
